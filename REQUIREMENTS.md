# Assignment compliance

Each minimum-delivery requirement from the brief, mapped to where it's met and how it's verified.
The same table is rendered live on the dashboard (`pipeline/agentic/dashboard.html`).

| Requirement | Met | Where | Verified by |
|---|---|---|---|
| **Functional core loop** - game starts | ✅ | Start screen, tap/space/click to begin | `src/game/Game.ts`, `src/ui/ui.ts` |
| - pick up collectibles as the parrot flies forward for a set distance | ✅ | Scrolling world + distance-check collision adds points | `Spawner.ts`, `Collision.ts` (unit-tested), `Scoring.ts` (unit-tested) |
| - game ends, showing the score | ✅ | End card with final score + replay + CTA | `Game.ts` (`end()`), `ui.ts` |
| **Controllable parrot - joystick OR keyboard** | ✅ (both) | Keyboard (arrows / WASD) and on-screen virtual joystick feed the same axis | `src/input/Input.ts` |
| **≥1 AI-generated collectible scoring points** | ✅ | `assets/sprites/fruit.png`, AI-generated, awards points on pickup | `verify:provenance` (AI origin + SHA-256), `Scoring.ts` |
| **≥1 AI-generated tree environment asset** | ✅ | `assets/sprites/tree.png`, AI-generated, scattered along the path | `verify:provenance`, `Spawner.ts` |

## How the AI assets are proven genuine

The fruit and tree are generated by a real AI image model (Pollinations / FLUX) through the
pipeline in `pipeline/agentic/`, graded by an AI vision judge, and recorded in
`assets/provenance.json` with the generator, the grading model, and each image's SHA-256.
`npm run verify:provenance` (part of `npm run ci`) re-fingerprints every shipped sprite and fails
if it wasn't AI-generated or was altered. See `AGENTIC.md`.

## Notes / decisions

- **The player parrot is the template's rigged 3D model** (real wing-flap animation) for game feel.
  The brief requires the *collectible* and *tree* to be AI-generated, not the parrot - so the
  collectible and tree are AI-generated, and the parrot uses the animated model. (The pipeline also
  generates an AI parrot sprite, kept for provenance, but the in-game player uses the rigged model.)
- Everything is verified automatically by `npm run ci` (unit tests → asset gate → provenance →
  build → build gate → headless e2e).
